# excel-incell-translate
> Excel/CSV表格翻译利器 | 支持单文件翻译 & 文件夹批量递归翻译 | 保留原格式/公式/批注 | 单元格内双语对照 | 双向语种自动判定

## ✨ 核心功能
1. **双模式翻译，适配不同场景**
   - 单文件模式：直接上传单个`.xlsx`/`.csv`文件，秒出翻译结果，适合临时翻译需求
   - 批量模式：传入包含表格的文件夹路径，自动递归遍历所有表格文件统一翻译，适合大量文件处理
2. **格式零损耗**：100%保留原表格的列宽、公式、批注、单元格样式，翻译后无需二次调整
3. **智能判定**：自动识别源语言（中英互译），无需手动指定语种
4. **幂等安全**：翻译结果输出到独立`output`文件夹，不会覆盖原始文件
在 **同一个单元格内** 对 Excel（`.xlsx`）做双向翻译：单元格变成两行——**第一行原文、第二行译文**（换行分隔，开启自动换行）。原文一字不删、不改，不新增列、不新增文件、不破坏公式。

## 功能特性

- **单元格内双语**：译文直接嵌在源单元格里（原文在上、译文在下），文件结构（列数、Sheet 布局）完全不变。
- **双向自动判定**：单元格含中文 → 补英文；含纯英文 → 补中文；中英混合 → 按字典双向处理。
- **幂等 / 增量安全**：重跑时，已翻译（`原文⏎译文`）的单元格自动跳过，只补未译部分，绝不重复翻译或叠加。
- **源零改动**：公式、纯数字、空单元格、测试用例编号、缩写、品牌/运营商名、人名一律保留原样。
- **判断性字符强制不翻译**：`PASS` / `FAIL` / `N/A` / `OK` / `NG` / `YES` / `NO` 等测试结论字符，即便字典里写了也跳过硬规则（脚本 `STATUS_SKIP` 强制生效）。
- **公式安全**：保存时设 `fullCalcOnLoad=True`，打开文件即自动重算，公式本身不被破坏。
- **字典外部化**：领域字典与表头映射统一由外部 JSON 传入，脚本不含任何项目专有数据，跨项目通用。

## 用法

### 模式 A：翻译模式（默认，需 `--dict`）

提供外部字典 JSON，对每个文本单元格现译现嵌：

```bash
python scripts/embed_incell.py workbook.xlsx --dict dict.json [--backup clean.xlsx]
```

### 模式 B：恢复模式（从右侧错位列回收，必须带 `--backup`）

适用于上一次误把译文放到了右侧独立列（`后面的表格`），现需把译文回收进源单元格：

```bash
python scripts/embed_incell.py workbook.xlsx --recover --backup clean.xlsx
```

> 恢复模式**必须**显式提供 `--backup`，否则拒绝执行——避免误判列数翻倍而删掉真实数据列。

## 字典格式

表头与正文**共用同一份** `{ "原文": "译文" }` JSON。方向由字典条目本身决定（中文键→英文值，英文键→中文值）。

```json
{
  "Call": "通话",
  "Pass": "通过",
  "通话": "Call",
  "Wi-Fi Speed Test": "Wi-Fi 速率测试",
  "DUT": "被测设备"
}
```

规则：
- 不可译项（测试 ID、缩写、品牌/运营商名、人名、型号）**不要**写进字典，或映射为自身，脚本遇到无译文即跳过。
- 判断性字符（`PASS`/`FAIL`/`N/A`/`OK`/`NG`…）**不要**作为可译项写入——`STATUS_SKIP` 会强制跳过，即便写了也无效。
- 宁可漏译，不要错译（用户明确要求「不要乱写」）。

字典构建流程见 [`references/translation_method.md`](references/translation_method.md)。

## 工作原理（要点）

1. 源文件**先备份**，再以双模式加载：正常模式保留格式写入译文，`data_only=True` 模式读取源值。
2. 逐单元格判定方向 → 查外部字典 → 写入 `原文\n译文` 并开启 `wrap_text`。
3. 合并单元格非左上角只读，写入前跳过；公式/数字/编号跳过。
4. 已译单元格（`is_already_translated` 启发式：≥2 行且首尾语言相反）直接跳过，保证幂等。
5. 保存前设 `fullCalcOnLoad=True`，并将还原后的验证与干净备份逐格比对（对比公式串与静态值，非缓存值）。

## 已知坑点（开发前必读）

完整清单见 [`SKILL.md`](SKILL.md) 的 *Critical Pitfalls*：

- 禁用 `insert_cols`（会静默破坏源数据）→ 直接写现有单元格，清理用 `delete_cols`。
- `MergedCell.value` 只读 → 写前检查并跳过非左上角合并格。
- 公式保存后缓存值丢失 → 设 `fullCalcOnLoad=True`，验证比对「公式串+静态值」而非缓存值。
- 恢复模式不可靠猜列数翻倍（`mm == 2*bm`）→ 必须显式 `--recover --backup`。
- 判断性字符不翻译 → `STATUS_SKIP` 硬规则，优先级高于字典。
